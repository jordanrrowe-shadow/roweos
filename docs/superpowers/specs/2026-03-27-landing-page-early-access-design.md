## 3. Auto-Email Access Key Flow

**Trigger:** New account creation (email/password `createUserWithEmailAndPassword` or first-time social auth)

**Sequence:**
1. User creates account (any auth method)
2. Firebase `onAuthStateChanged` fires with new user
3. Check Firestore: does `roweos_users/{uid}` already have an `accessKey`?
4. If no existing key:
   a. Generate key: `ROWE-XXXX-XXXX` format (random alphanumeric, uppercase)
   b. Write to Firestore `access_keys/{key}` document:
      - `key`: the generated key string
      - `email`: user's email
      - `status`: 'active'
      - `tier`: 'solo' (default)
      - `createdAt`: server timestamp
      - `autoGenerated`: true
   c. Write key reference to `roweos_users/{uid}`:
      - `accessKey`: the generated key string
      - `accessKeyAutoGenerated`: true
   d. Send welcome email via `/api/resend-welcome`:
      - To: user's email
      - From: roweos@therowecollection.com
      - BCC: jordan@therowecollection.com
      - Subject: "Welcome to RoweOS Early Access -- Your Access Key"
      - Body: existing `generateBetaWelcomeEmail()` template (update "Beta" to "Early Access")
      - Include the generated ROWE-XXXX-XXXX key
5. Show inline verification screen (step 4 below)

**Key generation format:**
```
ROWE-XXXX-XXXX
where X = random from [A-Z, 0-9]
```

**Function location:** New function `autoGenerateAccessKey(user)` called from the auth state handler when a new user is detected without an existing key.
