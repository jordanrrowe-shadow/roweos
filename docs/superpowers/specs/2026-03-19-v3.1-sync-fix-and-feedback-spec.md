## Problem Statement

The V3 write-through sync architecture (v25.0) left V2-era safety guards in place that actively sabotage the cloud-authoritative design. This causes a critical data resurrection bug: deleted items (Pulse goals, Focus categories, Automations, Clients) reappear across devices because stale local data overwrites correct cloud state. Additionally, there are ~35 non-sync feedback items across all features that need resolution.

## Architecture: Cloud-Authoritative + Timestamp Merge

**Principle:** Firestore is the single source of truth. Every save writes to localStorage (instant UI) AND Firestore (immediate, non-blocking) in parallel. On pull/startup/listener, cloud data always wins. For item-level conflicts (same item edited on two devices), `_modifiedAt` timestamps determine the winner. Items that exist locally but not in cloud are treated as deleted (never resurrected), UNLESS they were created after the last successful sync (genuinely new offline items).

**Tech Stack:** Firebase Firestore v10.7.1 compat SDK, vanilla ES5 JavaScript, single-file HTML app (`RoweOS/dist/index.html`, ~174K lines)

**Critical Constraints:**
- ES5 only (no arrow functions, let/const, template literals)
- All edits in `RoweOS/dist/index.html`
- `shouldSyncCategory()` gating MUST be preserved on all write paths
- Never add `margin: 0` to `* {}` reset
- Never remove iOS box-sizing reflow hacks
- `#agentConversation` must use `overflow: hidden`
- Never use em-dashes in UI text

## PHASE 1: Critical Sync Fix (5 bug vectors)

### 1.1 Fix `safeSyncWrite` -- Cloud is Authoritative

**File:** `index.html` ~line 129210
**Bug Vector:** When cloud returns `[]` or `{}`, the function refuses to overwrite local data, preserving stale/deleted items.

**Fix:** Remove the "skip empty cloud overwrite" guard entirely.

```javascript
function safeSyncWrite(key, cloudData) {
  if (cloudData === null || cloudData === undefined) return;
  var cloudStr = typeof cloudData === 'string' ? cloudData : JSON.stringify(cloudData);
  // v25.2: Cloud is authoritative -- always apply, even if empty.
  // Empty cloud data means it was intentionally deleted/cleared.
  localStorage.setItem(key, cloudStr);
}
```

### 1.2 Fix `manualSyncNow` -- Pull Only, No Blind Push

**File:** `index.html` ~line 159528
**Bug Vector:** Pushes ALL stale local data to Firestore BEFORE pulling, overwriting correct cloud state.

**Fix:** Remove the push phase. Write-through already handles all pushes on every save action. `enablePersistence` handles offline queue.

```javascript
function manualSyncNow() {
  if (typeof loadFromFirebaseV2 !== 'function' || !firebaseUser) {
    showToast('Firebase not connected', 'error');
    return;
  }
  showToast('Syncing...', 'info');
  // v25.2: Cloud-authoritative -- pull only.
  // Write-through already pushes on every save. enablePersistence handles offline queue.
  loadFromFirebaseV2(true).then(function() {
    var syncTime = String(Date.now());
    var syncDevice = typeof getDeviceType === 'function' ? getDeviceType() : 'unknown';
    localStorage.setItem('roweos_last_sync', syncTime);
    localStorage.setItem('roweos_last_sync_device', syncDevice);
    if (typeof updateLastSyncDisplay === 'function') updateLastSyncDisplay(new Date().toLocaleString(), syncDevice);
    if (typeof updateSyncHubStatus === 'function') updateSyncHubStatus();
    return new Promise(function(resolve) { setTimeout(resolve, 1000); });
  }).then(function() {
    if (typeof renderSyncInventory === 'function') return renderSyncInventory();
  }).then(function() {
    if (typeof refreshStorageDisplays === 'function') refreshStorageDisplays();
    showToast('Sync complete', 'success');
  }).catch(function(e) {
    console.error('[manualSyncNow] Error:', e);
    showToast('Sync failed: ' + (e.message || e), 'error');
  });
}
```

### 1.3 Fix `reconcileOnStartup` -- Always Pull

**File:** `index.html` ~line 60710
**Bug Vector:** Timestamp comparison is unreliable; "local cache is current" early exit prevents pulling authoritative cloud data.

**Fix:** Always pull from cloud on startup.

```javascript
function reconcileOnStartup() {
  if (!firebaseUser) return;
  console.log('[Sync V3.1] Startup -- pulling from cloud (cloud-authoritative)');
  if (typeof loadFromFirebaseV2 === 'function') loadFromFirebaseV2();
}
```

### 1.4 Disable `migrateToSyncV3` Push

**File:** `index.html` ~line 60561
**Bug Vector:** Each device independently re-uploads all local data on first V3 load, resurrecting items deleted on other devices.

**Fix:** Make it a no-op that just sets the migration flag.

```javascript
function migrateToSyncV3() {
  if (!localStorage.getItem('roweos_sync_v3_migrated')) {
    localStorage.setItem('roweos_sync_v3_migrated', 'true');
    console.log('[Sync V3.1] Migration flag set (cloud-authoritative, no push needed)');
  }
}
```

### 1.5 Fix Real-Time Listeners -- Handle Empty/Deleted State

**Files:** `index.html` ~lines 123736 (automations), 123760 (pulse)
**Bug Vector:** Listeners call `safeSyncWrite` which (before fix 1.1) blocked empty data. Even after fix 1.1, listeners need to handle `doc.exists === false` (document deleted from Firestore).

**Fix:** Add explicit handling for deleted documents in all listeners.

For Pulse listener:
```javascript
var unsubPulse = db.doc(basePath + '/pulse/main').onSnapshot(function(doc) {
  if (doc.metadata.hasPendingWrites) return;
  if (!shouldSyncCategory('goals')) return;
  if (!doc.exists) {
    // v25.2: Document deleted from cloud -- clear local
    localStorage.setItem('roweos_pulse_goals', '[]');
    localStorage.setItem('roweos_pulse_journal', '[]');
    pulseGoals = [];
    if (typeof renderPulseGoals === 'function') renderPulseGoals();
    return;
  }
  var pulse = doc.data();
  if (pulse.goals !== undefined) safeSyncWrite('roweos_pulse_goals', pulse.goals);
  if (pulse.journal) safeSyncWrite('roweos_pulse_journal', pulse.journal);
  if (pulse.entries) safeSyncWrite('roweos_pulse2_entries', pulse.entries);
  if (pulse.reminders) safeSyncWrite('roweos_reminders', pulse.reminders);
  // v25.2: Reload in-memory goals from what was just written
  try { pulseGoals = JSON.parse(localStorage.getItem('roweos_pulse_goals') || '[]'); } catch(e) {}
  if (typeof renderPulseGoals === 'function') renderPulseGoals();
  console.log('[Firebase V3.1] Pulse real-time update');
}, function() {});
```

For Automations listener:
```javascript
var unsubAutomations = db.collection(basePath + '/automations').onSnapshot(function(snapshot) {
  if (snapshot.metadata.hasPendingWrites) return;
  if (!shouldSyncCategory('automations')) return;
  var cloudAutos = [];
  snapshot.forEach(function(doc) { cloudAutos.push(doc.data()); });
  // v25.2: Cloud-authoritative -- always apply, even if empty
  safeSyncWrite('roweos_automations', cloudAutos);
  // Rebuild scheduled tasks
  try {
    var existingTasks = JSON.parse(localStorage.getItem('roweos_scheduled_tasks') || '[]');
    var nonAutoTasks = existingTasks.filter(function(t) { return t.type !== 'workflow' && t.type !== 'pipeline'; });
    var autoTasks = cloudAutos.filter(function(a) { return a.enabled !== false; }).map(function(a) {
      var full = JSON.parse(JSON.stringify(a));
      if (!full.type) full.type = 'workflow';
      if (full.enabled === undefined) full.enabled = true;
      return full;
    });
    localStorage.setItem('roweos_scheduled_tasks', JSON.stringify(nonAutoTasks.concat(autoTasks)));
  } catch(e) {}
  if (typeof renderAutomationsList === 'function') renderAutomationsList();
  console.log('[Firebase V3.1] Automations real-time update:', cloudAutos.length);
}, function() {});
```

## PHASE 2: Timestamp Conflict Resolution

### 2.1 Global `mergeByTimestamp` Utility

Place near sync globals (~line 60700 area).

```javascript
// v25.2: Timestamp-aware merge for array-based data
// Cloud is authoritative baseline. Local items only win if _modifiedAt is newer.
// Items that exist locally but NOT in cloud are treated as DELETED (not resurrected),
// UNLESS created after last successful sync (genuinely new offline items).
function mergeByTimestamp(localItems, cloudItems, idField) {
  var merged = {};
  var lastSync = 0;
  try { lastSync = parseInt(localStorage.getItem('roweos_last_sync') || '0'); } catch(e) {}

  // Cloud items are the authoritative baseline
  (cloudItems || []).forEach(function(item) {
    var key = item[idField] || item.name || JSON.stringify(item);
    merged[key] = item;
  });

  // Local items: only keep if (a) newer than cloud version or (b) created after last sync
  (localItems || []).forEach(function(item) {
    var key = item[idField] || item.name || JSON.stringify(item);
    if (merged[key]) {
      // Item exists in both -- newer _modifiedAt wins
      var cloudTs = merged[key]._modifiedAt || 0;
      var localTs = item._modifiedAt || 0;
      if (localTs > cloudTs) {
        merged[key] = item;
      }
    } else {
      // Item exists locally but NOT in cloud
      // Only keep if genuinely created offline after last sync
      var createdAt = item._modifiedAt || item._createdAt || 0;
      if (createdAt > lastSync && lastSync > 0) {
        merged[key] = item; // Genuinely new offline item
      }
      // else: was deleted on another device -- do NOT resurrect
    }
  });

  var result = [];
  Object.keys(merged).forEach(function(k) { result.push(merged[k]); });
  return result;
}
```

### 2.2 Stamp `_modifiedAt` on All Save Functions

Add `_modifiedAt = Date.now()` to each item on every save:

Example for `savePulseGoals`:
```javascript
function savePulseGoals() {
  // v25.2: Stamp _modifiedAt on modified goals
  var now = Date.now();
  pulseGoals.forEach(function(g) {
    if (!g._modifiedAt) g._modifiedAt = now;
  });
  localStorage.setItem('roweos_pulse_goals', JSON.stringify(pulseGoals));
  writeDB('pulse/main', { goals: pulseGoals }, { category: 'goals' });
}
```

Note: Only stamp goals that don't already have `_modifiedAt` in `savePulseGoals`. Individual mutation functions (add goal, edit goal, complete goal, delete goal) should stamp the specific item they modify.
